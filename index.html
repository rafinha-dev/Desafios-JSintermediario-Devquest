Exercício acordeon

figma

Construi o exercício fazendo uma div contendo somente os elementos no cabeçalho de cada questão, uma div para a div de cabeçalho e a resposta separadamente dentro dela.
Esse modelo fez com que eu não pudese usar overflow hiden pra esconder a resposta setando um tamanho para o box. O que fiz foi juntar a animação com opacidade e margin -40px para ele subir e desaparecer. A chamada da animação ficou dentro da classe que é aplicada com javascript.
A animação poderia ter sido feita com overflow hiden ao invez de animation poderia ter colocado apenas o transition, muito mais simples. No JavaScript a primeira vez coloquei a chamada manualmente com numeração em cada uma delas como parâmetro, poderia ser um listner com data[class] contendo o número da div e fazendo um arrayzão, também é possível que entre na estrutura indicando onde irá encolher a resposta dessa forma usando borbulhamento. Outra forma menos comum poderia ser usar o checkbox radio ou checkbox também, talvez usando js só para aplicar a classe no elemento marcado. Só não sei se seria semanticamente viável.

No caso de usar uma div única para a resposta encolher com overflow hiden, poderia usar o método element.classList.toggle("hide"). O toggle é usado quando uma classe pode ser usada com classlist add e remove. Ele valida entre true e false encurtando o código e evitando ter que fazer todo esquema de add e remove, porém a mesma tem que funcionar pra adicionar e remover se for mais de uma teria que ser o replace.

Ainda no JavaScript quando usado queryselectorall não pode usar um listner nele, logo para que seja possível, tem que usar o data. ou colocar um listner dentro de um foreach como elemento.

Por fim coloquei um código alternativo novo. O primeiro eu fiz com chamadas dentro do HTML como explicado acima, já o segundo consegui usar o listner fazendo um array e iterando dentro dos elementos com um lister dentro do forEach. Com o indice informado pelo forEach foi possível entrar dentro da div certa para fazer a modificação evitando ter que passar dentro de todo o código graças ao callback.

Exercício carrosel

Figma

*Fazer minha prórpria solução.

Nâo fiz minha prórpria solução sozinho, vou procurar mais informações e modelos de carroseis para fazer. Este foi criado com position absolute sobrepondo todas as divs em um mesmo lugar, usando opacity para passar por cada uma delas já que estão sobrepostas mesmo.
Os botões sobrepões todas as imagens e são os mesmos do inicio ao fim, diferente das divs mostradas abaixo deles, e essa foi minha principal dificuldade, pois tentei criar um botão para cada div e toda vez que trocava neles também trocava o botão. A solução para isso foi colocar os botões no topo fora de todas as divs, com position absolute.
A lógica dos botões é bem simples, já que o array vai de 0 a 3 e só temos 4 imagens, caso passasse para a imagem 4 estaria vazio, pois receberia opacity 0. Foi colocado um if com -1 só recebendo 3 cliques cada botão. Cada um deles tem um listner diferente. Enquanto um adiciona +1 no array para adicionar a classe o outro recebe -1. E caso o indice tenha chegado ao final é chamada uma função que tira a opacidade da setinha indicando que chegou ao fim.

Exercício transição de input

Este exercício foi muito interessante. A maneira que fiz foi colocar um listner para cada elemento, o que não é errado mas é uma duplicação tendo em vista que os mesmo irão monitorar a mesma mudança em cada input, isso viola os princípios da ortogonalidade e cria código inútil dificultando a manutenção, apesar de ser um bloco pequeno poderia estar em uma solução bem maior.
O interessante dessa solução vem de já na solução pronta usar um listner dentro do forEach, isso faz com que o listner só capture e aplique a mudança no elemento que passou por alteração e permitindo que o lister seja usado, pois ele não funciona com arrays. O listner aqui pode ser usado com evento 'input' monitorando tudo, 'change' monitorando só a alteração e aplicando só ao terminar.

Aqui mais uma vez usei o classlist.add() e classlist.remove() para aplicar a mudança ao outline e o border. Porem poderia ter feito diretamente na alteração de style do elemento javascript, só daria mais trabalho para pesquisar qual a descrição de style usar para plicar a alteração.

Exercício multiplicação

Esete era um exercício somente pelo terminal usando console.log(), mas eu inclui uma interface e um botão com innerHTML dentro de uma div para colocar os resultados.
Foi presiso armazenar os valores a serem multiplicados dentro de um array e fazer uma iteração sobre o array, pode-se usar o for ou o forEach. Quando o múmero passar pode usar um console.log para que ele seja impresso. Já no front para que todos estejam disponiveis é preciso declarar uma variavel global com um array vazio e incluir o resultado dentro dela. Se ela for declarada vazia dentro do modulo que fará os calculos sempre exibirá somente o número pois será inicializada todas as vezes que a repetição entrar em andamento. Por fim use o innerHTML com o array.
